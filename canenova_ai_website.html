System Design
Four-Layer Architecture
01 🔵 Sensing Layer NIR + Temperature + Moisture sensors mounted inline ›
NIR spectrometer (900–1700nm) captures spectral fingerprint of sugarcane. Temperature and moisture sensors compensate for ambient variation. Industrial mounting bracket attaches to existing conveyor frame in under 2 hours.
02 🟡 Edge Processing Layer Industrial PC running real-time data pipeline ›
Fanless industrial PC (DIN-rail mounted) runs the Python-based data pipeline. Spectral data is pre-processed (normalisation, scatter correction) and fed to the AI model at 10Hz sample rate. Local SSD buffers 30 days of readings offline.
03 🟢 AI Model Layer PLS Regression + Random Forest ensemble ›
Partial Least Squares (PLS) and Random Forest models run in ensemble, combining spectral features with metadata for robust Pol prediction. Model achieves ±0.19 RMSE on validation set. Seasonal retraining pipeline updates weights at season start.
04 🔴 Integration Layer SCADA / Dashboard / Alert systems ›
Modbus TCP and OPC-UA adapters push Pol readings to plant DCS/SCADA every second. REST API available for custom integrations. Email/SMS alerts fire when Pol drops below configurable threshold. Web dashboard accessible from any device on mill network.

The Science
How CANENOVA AI Works

Five integrated stages transform raw NIR light into actionable Pol data in under one second.

1

NIR Light Hits the Cane

Near-infrared light (900–1700nm wavelength) is emitted by the sensor head mounted above the conveyor belt. The light penetrates several millimetres into the sugarcane without any contact or sample preparation.

🔦 NIR Spectrometer · Non-contact
2

Spectral Data Captured

The sensor captures the unique spectral fingerprint of the cane — different molecules (sucrose, water, fibre) absorb NIR at distinct wavelengths. The full spectrum (512 data points) is captured in under 100ms.

📊 512-point spectrum · 100ms capture
3

AI Model Processes the Spectrum

The edge PC pre-processes the spectrum (normalisation, Savitzky-Golay smoothing, multiplicative scatter correction) then feeds it to the ensemble model (PLS + Random Forest) trained on thousands of lab-verified samples.

🤖 PLS + Random Forest · Edge inference
4

Pol % Output Generated

The model outputs a Pol % prediction with a confidence interval. Readings are averaged over a 10-second rolling window to smooth momentary variation. Alerts trigger instantly if Pol drops below the configured threshold.

📈 ±0.2 Pol accuracy · 10s rolling average
5

SCADA Receives the Data

Every Pol reading is pushed to the plant SCADA via Modbus TCP (or OPC-UA). Operators see live readings on existing HMI screens with no changes to their workflow. Historical data is stored for season-end reporting.

🔗 Modbus TCP · OPC-UA · REST API
